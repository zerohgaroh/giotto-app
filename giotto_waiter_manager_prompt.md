# Giotto NFC Platform — Промпт: страницы официанта и менеджера

## Контекст проекта

Ресторан Giotto — итальянский ресторан среднего и высокого ценового сегмента.
Гостевая страница уже реализована. Требуется разработать две отдельные страницы:
— Страница официанта (PWA, личный смартфон)
— Страница менеджера (PWA, смартфон или планшет)

Обе страницы — часть единой дизайн-системы Giotto:
- Основной цвет: `#0D2B6B` (navy)
- Акцент: `#C8A96E` (gold)
- Фоны: `#FAF7F2` (cream), `#FFFFFF`
- Типографика: Playfair Display (заголовки) + Inter (интерфейс)
- Иконки: Lucide Icons, stroke 1.5px
- Реальное время: WebSocket (Socket.io)
- Авторизация: JWT + RBAC

# ЧАСТЬ 1 — СТРАНИЦА ОФИЦИАНТА

## Общее

Официант авторизуется по логину и паролю. После входа — попадает на список своих столов.
Работает с личного смартфона через PWA (прогрессивное веб-приложение).
Официанту доступны только столы, которые назначил ему менеджер.

---

## Страница 1.1 — Авторизация

### Внешний вид
- Логотип Giotto по центру, белый на navy-фоне
- Поле «Логин» и «Пароль»
- Кнопка «Войти» — primary стиль (navy фон, белый текст)
- Ниже мелко: «Свяжитесь с менеджером для получения доступа»

### Логика
- После успешной авторизации — JWT-токен сохраняется, редирект на главную официанта
- Неверный логин/пароль — встроенное сообщение об ошибке под полями, без alert
- Сессия сохраняется до конца смены (или до ручного выхода)

---

## Страница 1.2 — Главная: мои столы

### Структура
- Шапка: логотип Giotto слева, имя официанта справа, кнопка выхода (иконка)
- Заголовок раздела: «Мои столы» (Playfair Display, 22px)
- Сетка карточек столов: 2 колонки на мобайле

### Карточка стола
Каждая карточка отображает:
```
┌─────────────────────────┐
│  Стол 5          [badge]│  ← номер + статус-badge
│                         │
│  Марко Р.               │  ← имя назначенного официанта (себя)
│  ⏱ 00:42               │  ← таймер: сколько гость сидит
│                         │
│  [иконка-предупреждение] Ждёт официанта  ← если есть активный вызов
└─────────────────────────┘
```

Стили карточки:
- Фон: `#FAF7F2`
- Граница: `1px solid #D4D1CB`
- Скругление: `18px`
- Тень: `0 2px 12px rgba(13,43,107,0.08)`

Если есть активный вызов (статус «Ждёт официанта» или «Просит счёт»):
- Граница карточки меняется на `#C8A96E` (gold), толщина 2px
- Внутри появляется строка с иконкой колокольчика и текстом вызова
- Карточка слегка пульсирует (CSS-анимация `pulse`, не агрессивная)

Тап по карточке → переход на детальную страницу стола.

### Push-уведомления
При новом вызове с любого из своих столов — PWA push-уведомление на смартфон официанта.
Текст: «Стол [N] — [причина вызова]»
Тап по уведомлению → открывает детальную страницу этого стола.

---

## Страница 1.3 — Детальная страница стола

Открывается при тапе на карточку стола.

### Шапка страницы
- Кнопка «← Назад» слева
- По центру: «Стол [N]» + badge статуса
- Справа: таймер (сколько гость сидит за столом)

### Блок: Активные вызовы (если есть)
Отображается только при статусах «Ждёт официанта» или «Просит счёт».

```
┌─────────────────────────────────────┐
│ 🔔  Гость вызывает официанта        │
│     Причина: Вопрос                 │
│     14:32 · 3 мин назад             │
│                                     │
│  [Принято — иду]                    │  ← кнопка, gold-стиль
└─────────────────────────────────────┘
```

Кнопка **«Принято — иду»**:
- Убирает вызов из блока
- Статус стола меняется на «Занят»
- Фиксируется время реакции официанта (для аналитики менеджера)
- Если вызовов несколько — показываются все по очереди

### Блок: Общий счёт стола
Заголовок: «Счёт стола» (Playfair Display Italic, 18px)

Единый список всех позиций: заказы гостя через NFC + добавленные официантом.
Каждая позиция:
```
Tagliata di manzo × 1 ............. 2 800 ₸
  ↳ Прожарка: medium rare           [от гостя]

Acqua Panna × 2 .................... 560 ₸
                                    [добавил официант]
```

Внизу: **Итого: X ₸** (жирный, navy)

Пометка «от гостя» / «добавил официант» — мелкий caption, `#8C8880`.

### Кнопка «Добавить заказ»
- Secondary стиль (прозрачный фон, navy граница)
- Тап → переход на страницу 1.4 (меню + корзина)

### Кнопка «Все обслужил»
- Primary стиль, золотой фон `#C8A96E`, navy текст
- Иконка: галочка (check)
- Располагается внизу страницы, sticky (всегда видна при скролле)

Логика нажатия «Все обслужил»:
1. На экране гостя автоматически всплывает модальное окно с просьбой оставить отзыв
2. Статус стола не меняется автоматически (официант сам управляет)
3. Кнопка становится неактивной на 30 секунд после нажатия (защита от двойного тапа)

### Быстрые заметки на стол (необязательное поле)
Поле для текста внизу страницы: «Заметки (видны только вам)»
Примеры: «аллергия на орехи», «день рождения», «VIP»
Сохраняются только на время сессии стола.

---

## Страница 1.4 — Меню + корзина (для добавления от официанта)

### Поведение
Открывается поверх детальной страницы стола (bottom sheet или полная страница).
Официант выбирает блюда как в обычном меню — они идут в общий счёт текущего стола.

### Структура
- Шапка: «Добавить в счёт · Стол [N]» + кнопка закрыть (×)
- Категории меню (горизонтальный скролл, tab bar)
- Карточки блюд: фото, название, цена, кнопка «+»
- Снизу sticky: корзина-итог + кнопка «Добавить в счёт» (navy)

### Логика
- Нажатие «Добавить в счёт»:
  - Позиции добавляются в общий счёт стола
  - Гость видит их в своём счёте через WebSocket (реальное время)
  - Страница закрывается, возврат на детальную страницу стола

---

## Поток: отзыв гостя (инициируется официантом)

Когда официант нажимает «Все обслужил» → на экране гостя всплывает модалка:

```
┌───────────────────────────────────┐
│  Как вас обслужили?               │
│                                   │
│  ★ ★ ★ ★ ☆   (интерактивные)     │
│                                   │
│  [Поле для комментария]           │
│                                   │
│  [Отправить]    [Пропустить]      │
└───────────────────────────────────┘
```

- Модалка появляется через WebSocket без перезагрузки страницы гостя
- Если гость не взаимодействует 60 секунд — автоматически скрывается
- Отзыв сохраняется в системе, привязан к столу, официанту и дате

# ЧАСТЬ 2 — СТРАНИЦА МЕНЕДЖЕРА

## Общее

Менеджер авторизуется через отдельный аккаунт с расширенными правами.
Видит все столы заведения. Управляет официантами, меню, следит за залом в реальном времени.

---

## Страница 2.1 — Авторизация

Аналогична авторизации официанта, но ведёт в панель менеджера.
Визуально идентична, роль определяется на бэкенде по JWT.

---

## Страница 2.2 — Главная: все столы зала

### Навигация
Нижний таб-бар (bottom navigation) с разделами:
- 🏠 Зал (главная)
- 🗺 Схема зала
- 👥 Официанты
- 🍽 Меню

### Раздел «Зал» — карточки всех столов

Заголовок: «Зал» + счётчик активных вызовов справа (red badge).

Сетка: 2 колонки на мобайле, 3-4 на планшете.

### Карточка стола (менеджер видит все поля)
```
┌──────────────────────────────┐
│  Стол 3             [Занят] │
│                              │
│  👤 Марко Р.                │  ← назначенный официант
│  ⏱ 01:24                   │  ← таймер с момента посадки
│                              │
│  🔔 Вызов: вопрос · 14:34  │  ← если есть активный вызов
└──────────────────────────────┘
```

### Анимация и звук при вызове официанта

Когда с любого стола приходит вызов (гость нажал «Позвать официанта» или «Принести счёт»):

**Анимация карточки:**
```css
@keyframes shake {
  0%, 100% { transform: translateX(0); }
  10%, 50%, 90% { transform: translateX(-6px); }
  30%, 70% { transform: translateX(6px); }
}
animation: shake 0.6s ease;
```
Анимация повторяется 3 раза, затем карточка остаётся с gold-границей.

**Звук:**
- Короткий мягкий звуковой сигнал (не агрессивный, под стиль заведения)
- Воспроизводится через Web Audio API
- Менеджер может отключить звук в настройках сессии

**После обработки:**
- Когда официант нажал «Принято — иду» — анимация останавливается, карточка возвращается к обычному виду

### Тап по карточке → детали стола (менеджер)
- Назначенный официант (с возможностью переназначить)
- Полный список заказов стола
- История вызовов за текущую сессию (время вызова / время реакции)
- Таймер нахождения гостя
- Кнопка «Освободить стол» — сбрасывает сессию, статус → Свободен

---

## Страница 2.3 — Конструктор зала

### Назначение
Визуальная карта зала — для удобства менеджера. На логику системы и привязку столов к официантам не влияет.

### Холст
- Белый холст с лёгкой сеткой (grid dots, `#D4D1CB`)
- Масштабируемый (pinch-to-zoom на мобайле)
- Перемещение по холсту (pan)

### Инструменты (панель снизу или справа)
- **Добавить стол** — иконка «+», выбор формы (квадрат / круг / прямоугольник)
- **Переместить** — drag-and-drop любого стола по холсту
- **Подпись** — тап по столу → редактировать номер/название
- **Удалить** — долгий тап → кнопка удаления
- **Добавить зону** — прямоугольник-область (например: «Терраса», «Основной зал», «Бар»)
- **Сохранить** — кнопка фиксирует текущую схему

### Элементы на холсте
Стол-квадрат:
```
┌─────┐
│  5  │  ← номер стола
│ ██  │  ← цвет = текущий статус (синхронизировано с реальным временем)
└─────┘
```
Цвет стола на схеме соответствует статусу (такой же как в карточках).
Если вызов — стол на схеме тоже трясётся и подсвечивается gold.

### Сохранение
Схема сохраняется в базе данных, привязана к аккаунту заведения.
При каждом открытии конструктора — загружается сохранённая схема.

## Страница 2.4 — Управление официантами

### Список официантов
Каждый официант — карточка:
```
┌────────────────────────────────────┐
│  Марко Р.           [Активен]      │
│  Столы: 1, 2, 3, 7                │
│                         [Изменить] │
└────────────────────────────────────┘
```

### Добавить официанта
Форма: имя, логин, пароль, выбор столов.

### Редактировать официанта
- Изменить имя, пароль
- Переназначить столы (мультиселект с номерами столов)
- Деактивировать аккаунт (официант не может войти)

### Привязка столов
- Каждый стол может быть назначен только одному официанту
- Если стол не назначен никому — он отображается у менеджера с пометкой «Без официанта»
- Менеджер может переназначить стол в любой момент, в том числе прямо из карточки стола

---

## Страница 2.5 — Управление меню

### Список блюд по категориям
Каждое блюдо — строка с:
- Фото (thumbnail 48×48)
- Название + категория
- Цена
- Переключатель «В стоп-листе» (toggle)
- Кнопка «Редактировать»

### Добавить / редактировать блюдо
Форма:
- Название (RU + IT)
- Категория (выпадающий список)
- Описание
- Состав / аллергены
- КБЖУ (4 поля)
- Цена
- Фото (загрузка)
- Активно / скрыто (toggle)

### Стоп-лист
Переключение «В стоп-листе» → блюдо мгновенно скрывается из гостевого меню через WebSocket. Никакой задержки, никакого обновления страницы у гостей.

---

## Дополнительная функция: история вызовов за смену

Доступна на главной странице менеджера (кнопка «История» в шапке).

Таблица / список:
```
14:32  Стол 3 · Вызов официанта · Причина: вопрос · Реакция: 2 мин
14:15  Стол 7 · Просит счёт · Реакция: 1 мин
13:58  Стол 1 · Вызов официанта · Причина: другое · Реакция: 4 мин
```

Фильтр по официанту, по столу, по типу вызова.
Данные за текущую смену (или выбранный день).

# ТЕХНИЧЕСКИЕ ТРЕБОВАНИЯ

## WebSocket-события (реальное время)

| Событие | Отправитель | Получатели | Описание |
|---|---|---|---|
| `waiter:called` | Гость | Официант стола + менеджер | Вызов официанта |
| `bill:requested` | Гость | Официант стола + менеджер | Гость просит счёт |
| `waiter:acknowledged` | Официант | Гость + менеджер | Официант принял вызов |
| `waiter:done` | Официант | Гость | Официант обслужил, просьба оставить отзыв |
| `order:added_by_waiter` | Официант | Гость (счёт) | Официант добавил позиции в счёт |
| `review:submitted` | Гость | Менеджер | Гость оставил отзыв |
| `table:status_changed` | Система | Все подключённые | Смена статуса стола |
| `item:stop_listed` | Менеджер | Все гостевые страницы | Блюдо убрано из меню |

## Cooldown (защита от спама)

Гость может повторно нажать «Позвать официанта» или «Принести счёт» не раньше чем через **2 минуты**.
На кнопке отображается обратный отсчёт: «Повтор через 1:43».
После истечения кнопка снова активна.

## Авторизация и сессии

- JWT-токен, время жизни: 12 часов (одна смена)
- Роли: `waiter` | `manager`
- При истечении токена — редирект на страницу авторизации
- Менеджер может принудительно завершить сессию официанта

## Требования к PWA

- Service Worker для работы при нестабильном Wi-Fi
- Web Push Notifications (Push API) — уведомления официанту даже при свёрнутом браузере
- Иконка приложения, splash screen в стиле Giotto
- Запрет на зум страницы (`viewport: user-scalable=no`) — интерфейс оптимизирован под мобайл

# СЦЕНАРИИ ИСПОЛЬЗОВАНИЯ

**Официант принимает вызов:**
> Смартфон вибрирует, приходит push. Тапаю — открывается стол 5. Вижу: «Гость вызывает — вопрос». Нажимаю «Принято — иду». Иду к столу, помогаю. Возвращаюсь, нажимаю «Все обслужил». У гостя всплывает просьба оценить.

**Официант добавляет заказ:**
> Гость за столом 3 просит добавить воду устно. Открываю стол 3, тапаю «Добавить заказ», нахожу «Acqua Panna», добавляю × 2. Нажимаю «Добавить в счёт». Вода появляется в общем счёте — и у меня, и у гостя.

**Менеджер видит вызов:**
> На главной карточка стола 7 затряслась, звуковой сигнал. Вижу: «Просит счёт». Проверяю — Марко уже принял вызов, анимация остановилась. Всё в порядке.

**Менеджер меняет стоп-лист:**
> Заканчивается тирамису. Открываю меню, нахожу «Tiramisù», включаю «Стоп-лист». Блюдо мгновенно исчезает у всех гостей в меню. Никакой перезагрузки.

**Менеджер строит схему зала:**
> Открываю конструктор. Перетаскиваю столы как мне удобно, добавляю зоны «Терраса» и «Основной зал». Сохраняю. Теперь вижу схему вместо карточек — и на ней сразу видно где вызовы, где свободно.
